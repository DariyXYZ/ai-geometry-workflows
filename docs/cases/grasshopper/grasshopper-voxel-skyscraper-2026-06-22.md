## Goal

Create a seed-driven voxel skyscraper in Grasshopper that looks like a coherent
architectural tower, not a random point cloud.

## Accepted Result

The accepted result is the controlled voxel tower generated by the seed-mask
C# Script node:

```text
Seed / floors / grid / density / twist / taper / terrace bias
-> deterministic occupied voxel mask
-> voxel centers and fast envelope mesh
-> optional Pufferfish Voxel Mesh downstream
```

Observed live Grasshopper metrics:

```text
seed=240622
floors=48
height=187.2 m
grid=13 x 11
occupied voxels=2881
EnvelopeMesh=Mesh (V:11888 F:2972)
FloorOutlines=48
FacadeLines=1632
PluginGuides=13
```

The visual quality came from architectural constraints inside the mask:

- floor-by-floor section stack;
- taper and waist;
- total twist;
- core preservation;
- terrace carving;
- seed-driven edge breakup, not unconstrained random placement.

## What Worked

- Seed changes produce stable variants instead of re-randomizing on every solve.
- The form reads as a tower because occupancy is filtered by floor, radius,
  taper, twist, and terrace rules.
- `EnvelopeMesh` gives a fast preview.
- `VoxelPoints` can feed Pufferfish `Voxel Mesh`, keeping the actual voxel mesh
  generation visible in Grasshopper.
- `FloorOutlines`, `FacadeLines`, and `PluginGuides` keep downstream native and
  plugin workflows available.

## What Did Not Work

The pure node-only proof:

```text
Center Box
-> Populate 3D
-> Pufferfish Voxel Mesh
```

solved correctly, but the result was a random voxel cloud, not an architectural
skyscraper. It lacked:

- floor logic;
- tower mask;
- grounded base;
- taper/twist/terrace controls;
- facade or section hierarchy.

Do not present that branch as the accepted building result.

## Reusable Workflow

Use this workflow when the user wants a convincing voxel tower quickly:

```text
controlled seed-mask generator
-> output occupied voxel centers
-> Pufferfish Voxel Mesh for visible GH/plugin voxelization
-> Custom Preview
-> optional facade lines / guide curves / loft mesh after the massing reads well
```

If the user asks for a node-only recreation, first rebuild the architectural
mask with visible GH logic:

```text
Series floors
-> grid points per floor
-> height-normalized taper/twist/terrace expressions
-> random threshold by seed
-> Cull Pattern
-> Pufferfish Voxel Mesh
```

Do not substitute `Populate 3D` in a box unless the task is only a random voxel
cloud proof.

## Promoted Rules

- A seed-driven voxel tower needs a deterministic architectural mask, not just
  random point population.
- Pufferfish `Voxel Mesh` is a good downstream mesh builder when fed controlled
  voxel centers.
- Keep fast preview geometry separate from heavy presentation geometry.
- Mark the C# seed-mask version as accepted; mark the `Populate 3D` node-only
  branch as a rejected proof for architecture.
